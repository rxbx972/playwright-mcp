# 유효성 검사 관련 검증 누락 발생

### 1. 문제 개요

특정 테스트 케이스에서 유효성 검사 오류 관련 검증이 누락되는 현상이 반복적으로 발생함.

> 예시:
> 
> 
> “필수 입력값을 비운 채 저장 버튼을 클릭하면 에러 메시지가 노출된다”라는 테스트 케이스에서 실제로는 검증 로직이 실행되지 않음.
> 

### 2. 원인 분석

### 2.1 코드 분석

- 자연어 기반 테스트 실행 코드에서 특정 키워드가 포함되어야만 해당 검증 로직이 작동하도록 설계되어 있음.
- 인식 기준 키워드: 유효성 검사 결과

### 2.2 케이스 확인

- 아래 케이스들은 의미상 유효성 검사 검증을 의도했으나, 시스템이 인식하는 키워드와 일치하지 않아 로직이 실행되지 않음.

| **기존 테스트 케이스 문장** | **인식 실패 사**유 |
| --- | --- |
| “유효성 검사 오류 표시 아이콘이 4개 노출된다” | 핵심 키워드인 “유효성 검사 결과” 미포함 |
| “입력 필드 하단에 유효성 검사 문구가 표시된다” | 유사 키워드지만 시스템 정의 키워드와 불일치 |
| “입력창 하단에 빨간색으로 '이름은 한글과 영문만 입력 가능합니다' 문구 노출된다.” | 시각적 설명 위주, 키워드 없음 |

### 3. 개선 조치

### 3.1 키워드 명세 정의

- 테스트 자동화 시스템이 인식 가능한 표준 키워드로 다음과 같이 정의함:
    - 유효성 검사 오류
    - 사용 예시: "유효성 검사 오류 메시지가 노출된다: {validation_messages}"

### 3.2 케이스 점검 및 정비

- 기존 유효성 검사 관련 테스트 케이스를 전수 검토
- 아래와 같은 표준 문장으로 수정 진행:

```markdown
예: 
기존 → “입력창 하단에 빨간색 오류 문구 노출됨”  
수정 → “유효성 검사 오류 메시지가 노출된다: ‘이름은 한글과 영문만 입력 가능합니다’”
```
